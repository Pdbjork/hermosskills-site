# Guardian ECC Skill Policy

Use this skill when invoking ECC-derived skills installed with the `ecc-` prefix.

Installed selected ECC skills support brand/content/social, market research, investor materials/outreach, API/backend/frontend/security/testing, product capability planning, MCP server patterns, evals, verification, and agent debugging.

Guardrails:
- ECC skills are advisory patterns, not permission to bypass Guardian-Builder rules.
- Posting, cross-posting, X API action, DMs, email outreach, investor outreach, or any external message requires explicit human approval unless the user separately approved a specific send.
- Apply Stop Slop to public prose after ECC content skills.
- Chromebloom: no dropshipping/internal fulfillment language in customer-facing copy.
- ULoad: legal storage-unit rental + one helper only.
- Hermosskills: public-good AI skills, sponsorship, commissions, maintenance funding, human-benefit orientation.
- Offline Helper: local-first/private AI setup, plain-English boundaries, no impossible “100% offline” promise.
- Do not store or print secrets, API keys, tokens, bank credentials, 2FA codes, or full SSNs.

Excluded for now because they are duplicate, too specific, or require unavailable external services: bun-runtime, nextjs-turbopack, deep-research, exa-search, fal-ai-media, dmux-workflows, documentation-lookup, mle-workflow, strategic-compact, everything-claude-code, benchmark-methodology.
